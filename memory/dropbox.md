# Dropbox Memory

Split from `MEMORY.md` on 2026-07-02. Formal playbook: `DRS-DROPBOX-PLAYBOOK.md`.

## Safety

- Dropbox is readable, but do not modify anything in Dropbox without Dave's explicit permission each time.
- Treat Dropbox as a source system: read/browse first.
- Do not delete, move, rename, overwrite, reorganize, or create project folders unless explicitly approved/asked.
- Operations folders are sensitive/restricted; do not summarize or expose HR, legal, finance, payroll, owner compensation, credentials, or private admin material unless specifically authorized.

## Folder Model

- Projects: real DRS project folders, active/in-progress/completed.
- Prelims / Proposals: inquiries, pursuits, opportunities before they become full projects.
- Operations: HR/legal/office/admin material; restricted by default.
- Engineering: engineering templates and reference material.
- Design: CAD/design-related material.
- Active and completed projects may be mixed together; do not infer status based only on folder location.

## Naming And Filing

- Typical project folder naming: `YYYY-### Project Name - Client Name`, e.g. `2026-118 1028 Ridgedale Dr - Titan`.
- Common project subfolders: Contractor Submittals, Correspondence, Design, Drawings, Incoming Docs / Incoming, Outgoing, Photos, Preliminary Design, Proposal & Estimate, RFI, Soil Report, Submittals, Testing.
- Dropbox filing behaviour: identify the project first.
- If a destination subfolder is specified, use that existing location.
- Otherwise use the existing Incoming / incoming-docs folder.
- When filing into Dropbox, use dated subfolder format `YYYYMMDD Client Name`, e.g. `20260514 Titan`; ask if client name cannot be determined reliably.
- Search by project number first when available; otherwise search by address, client, contractor, or project name.
- Prefer existing structure; ask rather than guessing.
